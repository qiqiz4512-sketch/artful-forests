# Artful Forests 当前网站画面与功能报告

更新时间：2026-03-17

## 1. 产品当前形态概览

当前网站并不是传统多页面站点，而是“单路由 + 高交互森林场景”的沉浸式 Web 应用。

- 主业务路由只有 1 个：首页 /
- 兜底路由有 1 个：404 页面
- 绝大多数画面、交互、功能入口都集中在首页场景中，通过浮层、悬浮面板、弹窗、气泡、动态特效来承载

从 Gemini 的产品理解角度，可以把当前网站定义为：

1. 一个可横向探索的森林世界场景
2. 一个基于树人格设定的社交模拟系统
3. 一个带有 SecondMe 身份接入的“认领森林身份 + 手动播种 + 自动演化”体验站

## 2. 当前所有页面与画面

## 2.1 路由级页面

### A. 首页 /

这是当前唯一的核心业务页面，所有主要功能都在这里完成。

首页不是单纯静态落地页，而是包含以下可见画面层：

1. 全屏森林主场景
2. 顶部季节/天气/时间控制区
3. 顶部右侧登录风铃入口
4. 顶部生态分区标签
5. 顶部临时通知条与登录成功庆祝条
6. 右侧聊天面板
7. 左下树种图鉴面板
8. 左下背景音乐蘑菇入口
9. 右下播种按钮
10. 右下绘画播种面板
11. 登录弹窗
12. 鼠标跟随种植幽灵预览层
13. 树与树之间的能量连线/对话连线层
14. 各类季节、天气、飞鸟、粒子、天象庆典特效层

### B. 404 页面

用于访问不存在路径时显示。

当前内容比较简单：

- 显示 404 文案
- 提供返回首页链接
- 控制台输出错误路径日志

## 2.2 首页内的“子画面/浮层/弹层”

虽然当前路由只有一个首页，但从产品画面角度，首页内部已经具备多个可独立描述的子界面。

### 1. 森林主场景

这是用户进入站点后看到的主要画面。

画面构成：

- 远景/天空/山体/草地等分层背景
- 横向延展的森林世界
- 不同生态区的树木群落
- 树木本体、树木标签、树木气泡、树木身份卡
- 粒子、风、雨、雪、鸟、种子漂浮、极光/流星等动态效果

用户可直接在该场景中完成浏览、拖动、缩放、点树、种树、观察社交互动等操作。

### 2. 登录弹窗

当用户未登录，或触发需要身份的动作时，会弹出“认领你的森林通行证”弹窗。

当前特点：

- 仅保留 SecondMe 单点登录入口
- 支持错误提示
- 支持错误震动反馈
- 在强制鉴权状态下不可随意关闭
- 支持登录跳转中状态

### 3. 绘画播种面板

用户点击右下角种子按钮后打开。

这是一个独立小画板，用户先画树，再把画作播种进森林。

包含：

- 300x300 画布
- 三种笔刷：水彩、蜡笔、勾线
- 多种颜色选择
- 清空、取消、播种按钮
- 自动裁剪有效绘制区域并生成 PNG Data URL

### 4. 种植预览幽灵层

当用户完成绘制、进入待播种状态后，会出现跟随鼠标的树木幽灵预览。

特点：

- 鼠标移动时实时跟随
- 仅允许放置在草地/地面区间
- 点击后落地为手动种植树

### 5. 右侧聊天面板

这是首页中最完整的“信息流/对话”界面。

包含两种状态：

- 折叠态：细长木纹标签条，带未读提示
- 展开态：完整聊天/日志面板

展开后包含：

- 标题：森林絮语 / 神谕日记
- 消息类型筛选：全部、对话、史诗、诏令
- 会话时间分组
- 成组显示的树木对话卡片
- 热搜/史诗/诏令标签
- 点赞、评论等虚拟互动数据展示
- 底部输入框与发送按钮
- 面板拖拽、收起、调整高度能力

### 6. 树种图鉴面板

位于左下角，默认折叠。

展开后显示：

- 当前区域名
- 可见树种数
- 树木总数
- 树种排行榜（可见数量 / 总数）

本质上是场景的轻量数据概览面板。

### 7. 背景音乐蘑菇入口

位于左下区域，独立控制 BGM。

特点：

- 支持自动播放尝试
- 如果浏览器拦截自动播放，会等待用户交互后恢复
- 支持淡入淡出
- 冬季样式下带冰裂音效
- 春季可叠加随机鸟鸣

### 8. 顶部右侧风铃登录入口 / 个人中心入口

未登录时：

- 显示“登录”风铃按钮
- hover 时有“认领这片林子”提示

已登录时：

- 显示用户名缩写
- 点击打开小型个人菜单
- 当前仅支持退出登录

### 9. 顶部左侧环境控制浮层

包含三个独立控制器：

1. 季节选择：自动、春、夏、秋、冬
2. 天气选择：晴、雨、雪
3. 时间选择：自动、晨、昼、暮、夜

每个控制器都有：

- 当前状态胶囊按钮
- 下拉浮层
- 文案提示说明

### 10. 顶部通知与庆祝浮层

包括两类：

1. 通知条：例如新树发芽、登录提示、需要先登录、手动树降临等
2. 庆祝条：例如登录成功后的身份激活反馈

其中部分通知支持点击后聚焦对应树木。

### 11. 场景内树木身份卡与气泡

用户 hover 树木时，会出现身份卡。

身份卡内容包含：

- 树名称
- 人格符号
- 树所处成长阶段称号
- 话痨标记
- 一句风格化梗文案

树木在对话、被点击、记忆回想等状态下，也会出现不同形式的头顶气泡或提示。

### 12. 天象庆典层

当用户亲手种下一棵“神启之树”时，会触发短时强化视觉层。

当前已实现：

- 流星或极光二选一庆典特效
- 全局能量连接强化
- 场景肃静状态

## 3. 当前所有核心功能

## 3.1 场景浏览与镜头能力

首页主场景支持以下基础浏览能力：

- 按住拖拽，横向平移森林世界
- 鼠标滚轮缩放镜头
- 根据视口位置计算当前可见世界范围
- 在世界左右边缘显示“森林起点 / 远方林境”提示
- 仅渲染视口附近树木，进行基础裁剪优化

说明：这是一个横向大地图，而不是固定一屏的景观图。

## 3.2 初始森林生成

首次进入首页时，系统会自动生成基础森林。

当前策略：

- 生成一批聚簇式树群
- 再补一批前景锚点树
- 为每棵树生成名字、人格、形状、位置、层级等数据
- 让不同区域拥有不同树种分布倾向

这意味着首页初始不是空白，而是进入即有“已存在的森林社会”。

## 3.3 世界生态分区

当前森林世界被切成 3 个生态区：

1. 冷杉林带
2. 混交草甸
3. 花影暖林

这些分区会影响：

- 树种偏好
- 背景氛围色
- 粒子色相
- 聊天活跃度
- 邻居判定半径
- 生态区提示文案

用户当前能感知到的是：

- 顶部会显示当前所在生态区标签
- 不同区域树型与视觉氛围不同

## 3.4 季节、天气、时间系统

当前支持三套环境控制系统：

### 季节

- 自动按日期判断
- 手动切换春夏秋冬
- 影响树木滤镜与氛围
- 冬季自动将天气推向雪
- 影响自动播种时偏好的树型

### 天气

- 晴
- 雨
- 雪

影响粒子和气氛表现。

### 时间

- 自动跟随系统时间
- 手动切换晨、昼、暮、夜

影响整体主题配色与视觉基调。

## 3.5 背景特效系统

当前已经存在较完整的动态背景特效体系，包括：

- 通用粒子层
- 夏季时间特效
- 冬季季节特效
- 飞鸟与种子掠过效果
- 春季鸟鸣增强
- 极光/流星庆典效果
- 种子/羽毛漂流效果

这些特效不是纯装饰，它们和季节、生态区、天气、用户手动播种行为有关联。

## 3.6 SecondMe 登录与身份状态

当前登录系统以 SecondMe OAuth/SSO 为核心。

### 已实现能力

- 生成授权地址并跳转到 SecondMe
- 保存 OAuth state，回调时校验 state
- 解析 code / error / error_description 回调参数
- 开发环境走本地代理换 token
- 生产环境可调用 Supabase Edge Function 进行 code exchange
- 登录成功后把 session 存入 localStorage
- 页面启动时自动恢复本地有效会话
- 支持退出登录
- 登录错误映射为中文提示

### 当前登录后效果

- 获得 username
- 可关闭强制登录状态
- 可进行手动播种
- 显示个人风铃入口
- 弹出欢迎通知和庆祝条

### 当前限制

- 没有独立个人中心页面
- 没有用户资料编辑界面
- 没有多身份切换

## 3.7 手动播种功能

这是当前最强的用户主操作。

完整流程：

1. 点击右下种子按钮
2. 打开绘画播种面板
3. 在画布中画树
4. 点击“播种”
5. 进入鼠标跟随的待放置状态
6. 在草地区域点击落点
7. 生成一棵手动树并加入森林社会

手动树当前设定为：

- 神启人格
- 高能量初始值
- 由用户亲手种下
- 会触发全局肃静 + 神性 surge + 天象庆典
- 会成为后续与其他树互动的核心对象

这棵树本质上相当于“用户在森林中的化身/主理树”。

## 3.8 自动播种与森林自演化

除了用户手动种树，森林还会自动长出新树。

当前自动循环已默认开启，无显式 UI 开关。

### 自动长树来源分为两类

1. 外来种子
2. 树木繁育生出的幼苗

### 外来种子

- 定时检查后随机出现
- 生成陌生新树
- 支持优先生成在当前视野附近
- 会产生种子或羽毛漂流视觉反馈

### 繁育幼苗

- 需要高亲密度关系
- 需要成年树
- 不能近亲
- 伴侣关系需要匹配
- 满足条件后生成下一代树苗
- 自动继承部分人格/名字/形态特征
- 记录父母关系与家族关系

这部分让森林具备“自行繁衍”的社会进化感。

## 3.9 树木人格与社交模拟

当前站点的核心差异化在于：树不是静态装饰，而是带人格的代理体。

### 当前已存在的人格倾向

- 温柔
- 睿智
- 顽皮
- 活泼
- 社恐
- 神启

### 树木内部状态包含

- 名字
- 人格
- 能量
- 成长分数
- 社交状态
- 世代信息
- 父母信息
- 朋友/家人/伴侣关系
- 与其他树的亲密度
- 记忆主题与互动历史
- 说话节奏（话痨 / 普通 / 害羞）

## 3.10 树木自动对话 A2A

树和树会自动聊天，不需要用户手动触发。

当前逻辑包括：

- 根据邻居关系选择对话对象
- 根据生态区调节聊天频率
- 活泼/调皮树会尝试“捞沉默的树”
- 用户的手动树出现后，附近树会更容易对它产生“朝拜/关注”式对话
- 会根据关系类型生成不同语气：陌生、朋友、家人、伴侣
- 会拼接记忆线索、回声文本、剧情 beat、生态尾句
- 会产生虚拟点赞/评论/热搜等社交数据

用户看到的结果就是：

- 场景内会有树木间动态连线
- 聊天面板会不断刷新新的对话卡片
- 有些对话会进入“史诗”或“热搜”状态

## 3.11 用户主动给树发消息

用户除了看树木自动交流，也可以主动参与。

当前流程：

1. 点击一棵树
2. 场景自动聚焦到该树
3. 右侧聊天面板激活输入框
4. 用户向该树发送消息
5. 系统写入聊天记录
6. 目标树稍后返回一条基于人格的回复

当前回复机制说明：

- 不是接外部大模型实时生成
- 是按人格模板进行轻量生成
- 但已经形成“用户-树”对话回路

## 3.12 树木聚焦与点击反馈

点击树木后当前会发生多种联动：

- 镜头横向平滑聚焦到树所在位置
- 树高亮
- 聊天面板展开
- 输入框自动聚焦
- 树可能出现“被点到”的吐槽气泡

这让树本体成为真正的交互入口，而不是摆件。

## 3.13 树木身份卡与成长展示

用户 hover 树木时可看到身份卡，当前可感知的信息有：

- 树名
- 人格符号
- 身份称号
- 特殊属性，如“话痨”
- 风格化梗句

此外树木还有成长阶段视觉概念：

- Seedling
- YoungTree
- GreatTree

成长分数来自对话内容长度/频率累计。

这部分意味着树木会被“交流”驱动成长，而不是只按时间成长。

## 3.14 树木关系系统

当前社交关系已经不只是普通聊天，而是具备结构化关系。

包括：

- 朋友
- 家人
- 伴侣
- 父母 / 子代
- 亲密度映射

已实现行为：

- 提升亲密度
- 达到阈值后成为朋友
- 达到更高阈值且满足条件时可成为伴侣
- 伴侣可以生育下一代
- 血缘关系会阻止错误繁育

## 3.15 记忆与回忆机制

当前树木不只是“即时说一句就忘”，还存在轻量记忆。

已实现：

- 记录最近互动主题
- 保存有限条互动历史
- 生成记忆 cue
- 对话时有概率触发“延续上一次话题”
- 树体可进入短时“记忆回想”视觉状态

这让自动对话比纯随机文本更像连续叙事。

## 3.16 聊天面板能力

右侧聊天面板当前已经是完整工具面板，而不是简单日志区。

已实现：

- 折叠 / 展开
- 未读数量提示
- 未读人格符号提示
- 可拖动位置
- 可调节高度
- 按消息类型筛选
- 对话分组展示
- 时间分组展示
- 点击消息后聚焦说话树
- 底部输入框直连当前选中树
- Enter 发送，Shift+Enter 换行

## 3.17 通知与事件反馈系统

当前站点存在较丰富的事件反馈。

包括：

- 登录成功欢迎提示
- 登录失败提示
- 需要先登录的权限提示
- 新生树发芽提示
- 外来种子降临提示
- 手动神树降临提示
- 闲置过久时，主理树会主动发一条“守望者提醒”

部分提示支持点击后跳转/聚焦到对应树。

## 3.18 音效与音乐反馈

当前不只是视觉交互，也有较多声音层。

包括：

- 背景音乐播放与淡入淡出
- 冬季蘑菇触发冰裂音效
- 春季随机鸟鸣
- 手动播种时木质音/铃音
- 登录风铃音效
- 聊天面板开合时的轻声效
- 用户首次手动种下主理树时的神性音效

## 3.19 图鉴与统计视图

当前已有一块轻量“数据看板”能力，即树种图鉴。

它提供：

- 当前总树数
- 当前可见树种数
- 前 8 个树种的可见/总量统计
- 当前生态区名称

这使首页不只是情绪化场景，也有一定“系统观察”能力。

## 4. 当前信息架构总结

如果把当前网站拆成产品模块，结构大致如下：

### A. 入口与身份

- 风铃登录入口
- SecondMe 登录弹窗
- 本地 session 恢复
- 退出登录

### B. 世界浏览

- 横向拖动
- 缩放
- 世界边缘提示
- 当前生态区提示

### C. 环境控制

- 季节切换
- 天气切换
- 时间切换
- 对应视觉/粒子/生态偏好变化

### D. 森林生成

- 初始树群生成
- 自动外来种子
- 自动繁育幼苗
- 用户手动画树播种

### E. 树木个体系统

- 树型/物种
- 名字/人格
- 成长阶段
- 身份卡
- 被点击反馈

### F. 社交模拟系统

- 自动树聊树
- 用户对树发消息
- 亲密度
- 朋友/伴侣/家族
- 记忆与回忆
- 热搜/史诗/诏令

### G. 界面辅助系统

- 聊天面板
- 树种图鉴
- 顶部通知
- 音乐蘑菇
- 天象庆典

## 5. 当前可见但未完全产品化的能力

有一些能力已经存在代码基础，但目前更偏“系统行为”，还不是显式的产品按钮。

包括：

- 自动播种循环虽已开启，但没有用户可见开关
- 叙事模式状态存在，但当前首页没有独立控制入口
- Divine surge、全局肃静等属于事件机制，不是用户日常显式开关
- 树木成长已存在分数和阶段，但缺少专门成长面板
- 关系网络已存在，但没有独立关系图界面

这些点适合 Gemini 后续作为“下一步可产品化能力”参考。

## 6. 当前网站的明显边界与缺口

从现状看，当前网站已经很有氛围和机制，但仍然是“单场景原型/体验站”而不是完整平台。

当前缺口包括：

1. 没有多业务页面结构
2. 没有独立的个人主页、树档案页、关系页、世界地图页
3. 用户发给树的消息回复仍是人格模板式，不是 LLM 实时生成
4. 没有后台管理台
5. 没有显式任务系统、成就系统、收藏系统
6. 登录后除播种外，用户身份价值还比较轻
7. 树木关系和成长虽然存在，但没有完整可视化面板
8. 聊天内容当前主要驻留前端状态，没有看到真正的持久化社区时间线 UI

## 7. 给 Gemini 的一句话总结

当前网站是一个“以森林为世界观、以人格化树木为角色、以手动播种和自动社交演化为核心体验、接入 SecondMe 身份系统的单页沉浸式交互原型”。

如果 Gemini 需要继续参考，建议它重点从以下几个方向理解现状：

1. 当前不是缺画面，而是缺页面级产品结构
2. 当前不是缺机制，而是缺把已有机制产品化、面板化、可视化
3. 当前最有价值的差异化资产是“树人格 + 森林社会 + 用户主理树 + 氛围化场景表现”
